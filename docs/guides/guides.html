Guides

Here are a few guides about the ressources coming with the Pygen package.

Sphinx

To build the documentation, you can move to the docs/ folder:

../pygen/
    docs/...
    pygen/...
    .gitignore
    cli.py
    LICENSE
    README.md
    requirements.txt
    setup.py
    TODO.md

and run the following command:

1$ sphinx-build doc-source docs/

Warning

docs/ should be built with the previous command, otherwise Github Pages won’t be able to read the files properly. A .nojekyll should also be added in the docs/ folder if it is not already present.

reStructuredText

The sphinx documentation is written using the reStructuredText .rst language (though it can also be written with the Markdown .md language).

Most useful syntax

  • Italic, bold and code text
    1*Italic text*
2**Bold text**
3``Inline text``

    Example:

    Italic text

    Bold text

    Inline text

  • Headings
     1Heading 1
 2=========
 3
 4Heading 2
 5---------
 6
 7Heading 3
 8*********
 9
10Heading 4
11+++++++++
  • Line separators
    1----------------------------------------------------------------------------------
  • Code blocks
    1.. code-block:: console
2    :linenos:
3
4    your code here

    Example:

    1your code here
  • Making a Glossary
     1.. glossary::
 2
 3    environment
 4        A structure where information about all documents under the root is
 5        saved, and used for cross-referencing.  The environment is pickled
 6        after the parsing stage, so that successive runs only need to read
 7        and parse new and changed documents.
 8
 9    source directory
10        The directory which, including its subdirectories, contains all
11        source files for one Sphinx project.

    Example:

    environment

    A structure where information about all documents under the root is saved, and used for cross-referencing. The environment is pickled after the parsing stage, so that successive runs only need to read and parse new and changed documents.

    source directory

    The directory which, including its subdirectories, contains all source files for one Sphinx project.

  • Making a Note
    1.. note::
2    This is a note.

    Example:

    Note

    This is a note.

  • Making a Warning
    1.. warning::
2    This is a warning.

    Example:

    Warning

    This is a warning.

Sphinx autobuild

Leave the docs/ folder, then run the documentation on a local server which listens to any changes to the doc and updates them live.

1$ cd ..
2
3$ sphinx-autobuild docs/ docs/build/html
